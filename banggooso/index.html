CONTEXT
03

방구석연구소는 어떤 플랫폼인가

심리테스트·밸런스 게임·퀴즈 등 인터랙티브 콘텐츠를 즐기고 SNS에 공유하는 B2C 플랫폼.

방구석연구소는 사용자가 콘텐츠를 즐기고 결과를 SNS에 공유하는 바이럴 루프를 핵심 성장 엔진으로 삼은 플랫폼입니다.

대형 콘텐츠 하나가 수십만 참여를 이끌고, 참여자의 공유가 다시 신규 유입을 만드는 구조. 콘텐츠 자체가 광고 채널이자 수익 자산이었습니다.

그러나 흥행 후 트래픽이 정체된 콘텐츠 자산이 방치되고, SNS 광고 의존도가 높았으며, 대규모 트래픽에 따른 운영 안정성 이슈가 잠재해 있었습니다.

PLATFORM OVERVIEW

CONTENT 심리테스트 밸런스 게임 · 퀴즈 · 방탈출
TRAFFIC Meta Ads Google · Pinterest · SNS 바이럴
REVENUE B2B 소싱 자체 광고 구좌 · 트래픽 수익화
콘텐츠 → 공유 → 유입의 바이럴 루프가 핵심.
이 루프를 외부 플랫폼으로 연결한 것이 B2B 협업 모델.
PROBLEM
04

트래픽은 있었지만 활용이 부족했습니다

방치된 자산, 단일 수익 구조, 운영 안정성. 세 가지 문제를 동시에 풀어야 했습니다.

PROBLEM 01
유휴 콘텐츠 자산
흥행했던 콘텐츠가 시간이 지나며 트래픽 정체. 제작비는 이미 투입되었으나 추가 수익이 발생하지 않는 상태. 콘텐츠 IP 자체가 방치되고 있었습니다.
PROBLEM 02
수익 구조 다각화 필요
SNS 광고 예산에 의존하는 유입 구조. 광고 단가 상승·알고리즘 변화에 취약. 트래픽을 다른 방식으로 수익화하는 구조가 필요했습니다.
PROBLEM 03
보안·운영 안정성
대규모 B2C 트래픽에서 F12 방어, XSS 인젝션 등 보안 이슈가 잠재. 운영 안정성 확보 없이는 B2B 협업 확장이 어렵습니다.
MAIN CASE · 카카오뱅크 B2B 협업

방구석연구소 × 카카오뱅크 ｜ 유휴 콘텐츠를 수익원으로

MAIN CASE · STRATEGY

검증된 트래픽을 다른 플랫폼에서 다시 살리다

콘텐츠 IP를 외부 플랫폼에 소싱해 추가 제작비 없이 새로운 사용자에게 도달.

SIGNAL
방치된 자산
과거 흥행했던 콘텐츠가 시간이 지나며 트래픽 정체. 제작비는 이미 투입되었으나 추가 수익이 발생하지 않는 상태.
"투입된 자산이 유휴 상태로 남는다"
HYPOTHESIS
다른 플랫폼에서 재활용 가능
검증된 콘텐츠 IP를 외부 플랫폼에 소싱하면 추가 제작비 없이 새로운 사용자에게 도달할 수 있다.
"끝난 콘텐츠가 아니라 미활용 자산"
STRATEGY
B2B 콘텐츠 소싱 모델
카카오뱅크 청소년 서비스(미니)에 콘텐츠를 소싱. 양사 모두에게 유익한 협업 모델로 설계.
"콘텐츠 IP가 B2B 협상 자산이 된다"
MAIN CASE · EXECUTION

기술 제약과 일정 변동 안에서 결과로 연결

4가지 핵심 의사결정으로 협업을 실행 가능한 구조로 만들었습니다.

ACTION 01
URL 보안 이슈를 OneLink로 해결
직접 URL 노출이 보안상 불가했던 상황. OneLink 딥링크 방식으로 우회해 양사 보안 정책을 모두 충족하는 기술 대안을 정리했습니다.
양사 보안 정책 동시 충족
ACTION 02
양사 1주 한정 리소스 안에서 순차 작업
양사 개발팀 모두 1주 한정 리소스. 동시 작업 불가능. 순차 작업 일정으로 재설계해 일정 충돌을 방지했습니다.
일정 충돌 없이 진행
ACTION 03
변동 발생 시 범위·비용 구조 재설계
일정 변동·추가 요구 발생 시 협업 범위와 비용 구조를 재협의해 프로젝트 목표를 유지했습니다.
프로젝트 목표 유지
ACTION 04
서비스 오픈까지 끝까지 리드
개발 일정·협업 범위·배포 순서를 재설계해 실제 서비스 오픈과 성과 달성까지 연결했습니다.

MAIN CASE · REFLECTION

이 협업에서 배운 것

LEARNED 01
검증된 콘텐츠는 끝난 자산이 아니라 다른 플랫폼에서 새 생명을 가질 수 있다
한 플랫폼에서 정체된 콘텐츠가 다른 플랫폼에서는 신규 자산이 됩니다. 방구석연구소의 콘텐츠 IP가 카카오뱅크 미니의 콘텐츠 문제를 풀었고, 동시에 방구석연구소의 수익 구조도 확장했습니다.
LEARNED 02
B2B 협업은 양사 제약을 모두 푸는 솔루션 설계가 핵심이다
기술 제약(URL 보안), 일정 제약(1주 리소스), 범위 제약(추가 요구)을 각각 해결하지 않으면 협업 자체가 무산됩니다. 제약을 피하는 게 아니라 제약 안에서 실행 가능한 설계를 만드는 것이 PM 역할이었습니다.
GOOD
  • 기술 제약을 협업 모델로 우회 (OneLink 딥링크)
  • 양사 모두에 윈윈 결과. 추가 제작비 0으로 새 채널 확보
LIMIT
  • 협업사(카카오뱅크) 측 전환 데이터 미공유로 정량 임팩트 파악 어려움
  • B2B 소싱 모델의 반복 가능성. 추가 파트너사 확장 전 검증 필요

REFLECTION

방구석연구소가 가르쳐준 것

LEARNED 01
PM의 책임 폭이 곧 제품의 안정성을 결정한다
기획·운영·마케팅·보안까지 시야를 넓히지 않으면 제품의 약한 고리는 결국 어디선가 드러납니다. 방구석연구소에서 보안 이슈를 PM이 직접 들고 가지 않았다면 B2B 협업 확장 자체가 어려웠을 것입니다.
LEARNED 02
검증된 트래픽은 다른 플랫폼에서도 자산이 된다
방구석연구소의 콘텐츠 IP는 카카오뱅크에서 신규 자산으로 작동했습니다. 트래픽을 쌓은 플랫폼의 가치는 단순 UV가 아니라, 검증된 콘텐츠 IP가 다른 맥락에서 재사용될 수 있다는 데 있습니다.
GOOD
  • 다층적 책임, 기획·운영·보안·글로벌까지
  • B2B 소싱 모델 첫 성공 사례. 추가 제작비 0으로 신규 채널 확보
